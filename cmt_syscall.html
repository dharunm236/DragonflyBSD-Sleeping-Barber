DragonFly BSD: Commands and System calls

DragonFly BSD, like other Unix-like operating systems, utilizes system calls to interact with the kernel and perform various operations.

Get Started

COMMANDS

Explore the fundamental commands for DragonFly BSD, a Unix-like operating system known for its scalability and performance.

cd (Change Directory)

Syntax: cd [directory_path]

Description: Changes the current working directory to the specified directory.

ls (List)

Syntax: ls [options] [directory_path]

Description: Lists the files and directories in the current or specified directory.

mkdir (Make Directory)

Syntax: mkdir [directory_name]

Description: Creates a new directory with the specified name.

rmdir (Remove Directory)

Syntax: rmdir [directory_name]

Description: Deletes the specified directory if it's empty.

rm (Remove)

Syntax: rm [options] [file_path]

Description: Deletes the specified file(s).

chmod (Change Mode)

Syntax: chmod [options] [mode] [file_path]

Description: Changes the permissions of a file or directory.

chown (Change Owner)

Syntax: chown [options] [user:group] [file_path]

Description: Changes the owner and group of a file or directory.

ps (Process Status)

Syntax: ps [options]

Description: Displays information about active processes.

kill

Syntax: kill [options] [process_ID]

Description: Terminates a specified process.

pwd (Print Working Directory)

Syntax: pwd

Description: Prints the current working directory.

echo

Description: Display a line of text.

Usage: echo [options] [string]

Example: echo "Hello, World!" - Displays "Hello, World!"

rmdir

Description: Remove empty directories.

Usage: rmdir [options] [directory]

Example: rmdir my_directory - Removes the directory named "my_directory" if it's empty.

UNIQUE COMMANDS

Explore the UNIQUE commands for DragonFly BSD, a Unix-like operating system known for its scalability and performance.

hammer

Description: Manage HAMMER filesystems.

Usage: hammer <subcommand> [options]

Example: hammer status - Displays the status of HAMMER filesystems.

dports

Description: Manage DragonFly BSD's ports collection.

Usage: dports <command> [options]

Example: dports install <portname> - Installs a port.

lockf

Description: Run a command with a lock on a file.

Usage: lockf <file> <command> [args]

Example: lockf /var/lock/mylockfile myscript.sh - Runs myscript.sh with a lock on /var/lock/mylockfile.

w

Description: Show who is logged on and what they are doing.

Usage: w [options]

Example: w - Displays a list of users currently logged in and their activity.

dfly-upgrade

Description: Upgrade DragonFly BSD to a newer version.

Usage: dfly-upgrade [options]

Example: dfly-upgrade -y - Upgrades the system automatically.

dump

Description: Backup file systems to tape or disk.

Usage: dump [options] <file system>

Example: dump -0u -f /backup/root.dump / - Performs a level 0 dump of the root file system.

dd

Description: Convert and copy a file.

Usage: dd [operands]

Example: dd if=/dev/zero of=/tmp/testfile bs=1M count=100 - Creates a 100 MB file filled with zeros.

rc.d

Description: Control system services.

Usage: service <service> <command>

Example: service sshd start - Starts the SSH daemon.

acpi

Description: ACPI control program.

Usage: acpi [options]

Example: acpi -t - Shows the thermal information of the system.

ipcs

Description: Report inter-process communication facilities status.

Usage: ipcs [options]

Example: ipcs -a - Displays all IPC facilities status.

service

Description: Manage system services.

Usage: service <service> <command>

Example: service ntpd restart - Restarts the NTP daemon.

ipcrm

Description: Remove IPC facilities.

Usage: ipcrm <type> <id>

Example: ipcrm -m 12345 - Removes the shared memory segment with ID 12345.

SYSTEM CALLS

Explore the fundamental system calls for DragonFly BSD, a Unix-like operating system known for its scalability and performance.

open()

Description: Opens or creates a file.

Usage: int open(const char *path, int oflag, ...);

Parameters:

  • path: Path to the file.
  • oflag: Access mode (e.g., O_RDONLY, O_WRONLY, O_RDWR).
  • mode: Permissions (optional, used with O_CREAT).

close()

Description: Closes an opened file descriptor, freeing up system resources associated with it.

Usage: int close(int fd);

Parameters:

  • fd: File descriptor to be closed.

read()

Description: Reads data from an open file descriptor into a buffer in memory.

Usage: ssize_t read(int fd, void *buf, size_t count);

Parameters:

  • fd: File descriptor to read from.
  • buf: Buffer where the read data will be stored.
  • count: Number of bytes to read.

write()

Description: Writes data from a buffer in memory to an open file descriptor.

Usage: ssize_t write(int fd, const void *buf, size_t count);

Parameters:

  • fd: File descriptor to write to.
  • buf: Buffer containing the data to be written.
  • count: Number of bytes to write.

fork()

Description: Creates a new process by duplicating the existing process. The new process is known as the child process, and it inherits the memory space and resources of the parent process.

Usage: pid_t fork(void);

Returns: PID of the child process in the parent, and 0 in the child.

exec()

Description: Replaces the current process with a new program. It loads the program into the current process's address space and begins its execution.

Usage: Various forms such as int execve(const char *pathname, char *const argv[], char *const envp[]);

Parameters:

  • pathname: Path to the executable file.
  • argv: Argument list for the new program.
  • envp: Environment variables for the new program.

wait()

Description: Suspends the execution of the calling process until one of its child processes terminates. It also retrieves the exit status of the terminated child process.

Usage: pid_t wait(int *wstatus);

Parameters:

  • wstatus: Pointer to an integer to store the exit status of the terminated child process.

kill()

Description: Sends a signal to a process or a group of processes, allowing for inter-process communication and control.

Usage: int kill(pid_t pid, int sig);

Parameters:

  • pid: Process ID of the target process.
  • sig: Signal to be sent.

stat()

Description: Retrieves information about a file, such as its size, permissions, and timestamps.

Usage: int stat(const char *pathname, struct stat *statbuf);

Parameters:

  • pathname: Path to the file.
  • statbuf: Buffer to store the file information.

chdir()

Description: Changes the current working directory.

Usage: int chdir(const char *path);

Parameters:

  • path: Path to the new working directory.

Example: chdir("/new/working/directory");

mkdir()

Description: Creates a new directory.

Usage: int mkdir(const char *pathname, mode_t mode);

Parameters:

  • pathname: Path to the new directory.
  • mode: Permissions for the new directory.

Example: mkdir("/new/directory", 0755);

rmdir()

Description: Removes an empty directory.

Usage: int rmdir(const char *pathname);

Parameters:

  • pathname: Path to the directory to remove.

UNIQUE SYSTEM CALLS

Explore the UNIQUE system calls for DragonFly BSD, a Unix-like operating system known for its scalability and performance.

clock_gettime()

Description: Retrieves the current time of the specified clock.

Usage: int clock_gettime(clockid_t clock_id, struct timespec *tp);

Parameters:

  • clock_id: Identifier of the clock (e.g., CLOCK_REALTIME).
  • tp: Pointer to a timespec structure where the time will be stored.

Example: clock_gettime(CLOCK_REALTIME, &ts);

shm_open()

Description: Opens or creates a POSIX shared memory object.

Usage: int shm_open(const char *name, int oflag, mode_t mode);

Parameters:

  • name: Name of the shared memory object.
  • oflag: Flags (e.g., O_CREAT).
  • mode: Permissions for the new shared memory object.

Example: int shm_fd = shm_open("/myshm", O_CREAT | O_RDWR, 0666);

shm_unlink()

Description: Removes a POSIX shared memory object.

Usage: int shm_unlink(const char *name);

Parameters:

  • name: Name of the shared memory object to remove.

Example: shm_unlink("/myshm");

ksem_post()

Description: Increments (unlocks) a POSIX semaphore.

Usage: int ksem_post(ksem_t id);

Parameters:

  • id: Semaphore identifier.

Example: ksem_post(sem_id);

vfork()

Description: Creates a new process, shares the address space with the parent until execve() or exit().

Usage: pid_t vfork(void);

Returns: 0 to the child process. The child's PID to the parent process.

sigqueue()

Description: Sends a signal and accompanying data to a specific process.

Usage: int sigqueue(pid_t pid, int sig, const union sigval value);

Parameters:

  • pid: Process ID to which the signal will be sent.
  • sig: Signal to send.
  • value: Union containing the accompanying data.

Example: sigqueue(pid, SIGUSR1, (union sigval) { .sival_int = 123 });

ksem_open()

Description: Opens or creates a named POSIX semaphore.

Usage: ksem_t ksem_open(const char *name, int oflag, mode_t mode, unsigned int value);

Parameters:

  • name: Name of the semaphore.
  • oflag: Flags (e.g., O_CREAT).
  • mode: Permissions for the new semaphore.
  • value: Initial value for the semaphore.

Example: ksem_t sem = ksem_open("/mysem", O_CREAT, 0644, 1);

ksem_unlink()

Description: Removes a named POSIX semaphore.

Usage: int ksem_unlink(const char *name);

Parameters:

  • name: Name of the semaphore to remove.

Example: ksem_unlink("/mysem");

ksem_post()

Description: Increments (unlocks) a POSIX semaphore.

Usage: int ksem_post(ksem_t id);

Parameters:

  • id: Semaphore identifier.

Example: ksem_post(sem_id);
